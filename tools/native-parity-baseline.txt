# native-parity-baseline.txt — the allowed-gap ratchet for the in-process
# libLLVM native backend (KIR Lane 1.5, docs/kir-design.md §"Lane 1.5").
#
# Each non-comment line is a fixture (relative path) that currently
# DIVERGES from the C-direct oracle under `--backend=native` — a known
# gap, not a regression. tools/test-backend-parity.sh (with
# TARGET_BACKEND=native NATIVE_PARITY_RATCHET=1) gates against this list:
#
#   - a fixture that fails parity but is NOT listed here => FAIL (a NEW
#     gap / regression — the burn-down went backwards).
#   - a listed fixture that now PASSES => OK, with a prompt to remove it
#     from this file (the ratchet tightens; a gap closed).
#   - listed == failing => OK at baseline.
#
# The flip to native-default (Lane 1.5) proceeds only when this file is
# EMPTY (full corpus parity). First measured 2026-06-09: 168 gaps / 513
# fixtures. Burn-down 1 (2026-06-10) closed 31: the missing-symbols family
# (24, `kai_assert_check` symbol) + a slice of unbound-register (const /
# prelude-fn-value inline + nested record sub-pattern binds + default-arm
# binders) + `mini_ledger` (build-failed-other, closed as a side effect of
# the assert fix) → 137 gaps.
#
# Burn-down 2 (2026-06-10) closed 55 (137 → 82), two root causes, both in
# the KIR lowering (stage2/compiler/kir_lower*.kai):
#   (1) UNARY MINUS aliased the binary `-`. `EUnop("-", a)` lowered to
#       `KPrim("-", [a])` → a one-arg call to the two-arg `kaix_sub`; the
#       missing operand read an uninitialised register, so EVERY negative
#       literal / `0 - x` diverged. Fix: `unop_prim` maps `-` → `neg`
#       (→ `kaix_neg`, the one-arg helper, mirror of emit_c's `kai_op_neg`).
#   (2) LOCALS-SHADOW-IMPORTS. `lower_var` consulted the EFn/prelude table
#       BEFORE the local-binder check, so a param/binder named like a stdlib
#       `pub fn` (`init`, `sum`, `head`, `acc`, …) lowered to a CLOSURE over
#       the stdlib thunk instead of the local read — the closure then
#       reached an arithmetic op as a non-Int operand. Fix: thread a
#       `locals` set (params/captures + every `let`/match binder, via
#       `ls_emit`) and check it first in `lower_var` / `lower_callee_dispatch`
#       (the #748/#749 rule, the KIR mirror of emit_c's `EmitCtx.lcs`).
# This was transverse: the locals-shadow bug hit fixtures across runtime-
# type-mismatch (26/27 of the family), output-mismatch, exit-code-mismatch,
# SIGSEGV, missing-symbols, and no-effect-handler — any local name colliding
# with a stdlib fn → 82 gaps. Issue #801 (stream carrier, PR #809) then
# added 6 new corpus fixtures (4 stream_* + demos/wc + ctor_field_effect_row)
# as gaps → 88. Issue #810 (#[unstable] transparency, PR #810) then added
# `attr_unstable_refine_narrow` (native SIGSEGV, C clean — a backend gap,
# not a regression) → 89.
#
# CROSS-PLATFORM NOTE: `list_helpers` + `list_zip3_scan` PASS native on
# macOS but SIGSEGV (139) on Linux. Burn-down 2 closed them locally; CI
# (the merge oracle, Linux) showed them still divergent, so they stay
# listed (+2 → 91). The lesson burn-down 1's retro flagged: the ratchet is
# validated on Linux — a macOS-only pass is NOT a closed gap.
#
# Burn-down 3 (2026-06-10) closed 9 (91 → 82), one root cause in the KIR
# match lowering (kir_lower_variant.kai + kir_lower_walk.kai):
#   SHARED-TAG SUB-DISCRIMINATION. A `match` arm sharing its top-level tag
#   with a sibling but discriminating on a payload sub-pattern
#   (`Exited(0)` vs `Exited(_)`, `Branch(Red, _)` vs `Branch(Black, _)`)
#   lowered to a `KSwitch` with TWO cases of the same i32 tag → `module
#   verify failed: Duplicate integer as switch case` (LLVM rejects it),
#   AND the second arm was dead (nothing tested the payload). Fix: group
#   arms by tag (`plan_arms` dedups), emit ONE switch case per tag, and
#   for a tag shared by N>1 arms route that case to a sub-fail-chain that
#   tests each arm's payload — `kai_eq_raw slot lit` for a literal slot,
#   `kai_tag_eq slot ctor_tag` for a nullary ctor / enum slot (mirror of
#   the oracle's `emit_pat_test_variant_subs`). The unique-tag cases keep
#   the bare O(1) switch jump. The whole `Duplicate integer as switch
#   case` family (18 fixtures) is GONE: 9 close outright; the other 9
#   (regex / json-unicode / map) now BUILD + RUN but DIVERGE on a
#   PRE-EXISTING bug the build error masked (regex char-class parse, json
#   `\uXXXX` decode, Call-param-mismatch), reclassified to their new
#   families. `kaix_tag_eq` is an additive runtime shim (runtime_llvm.c).
#   The two cross-platform Linux-only SIGSEGV gaps (`list_helpers`,
#   `list_zip3_scan`) stay listed — burn-down 3 does not touch their root
#   cause. → 82 gaps remain — see docs/native-parity-gaps.md.
#
# Burn-down 4 (2026-06-11) closed 5 (82 → 77), one root cause in the KIR
# lowering name supply (kir_lower.kai `ls_fresh_reg`):
#   SYNTHETIC-REGISTER NAMESPACE COLLISION. `ls_fresh_reg` minted temps
#   `t0`, `t1`, … in the SAME namespace as user-source value binders. A
#   program that names a `let` binder or a match field binder `t1`/`t2`
#   (stdlib `map.kai`'s `tree_rotate_left` binds `TNode(_,_,_,t2,t3)`;
#   `fx_basic.kai` binds `let t0 = …`) collided with a same-numbered
#   synthetic temp. The native backend (one alloca per register name,
#   KSlot the single type source — `collect_fn_specs` keeps the FIRST
#   slot seen) FUSED the two: a raw-slot synthetic `t2` (`tagof` → i32)
#   and a boxed user binder `t2` shared `%t2.addr`, so the call passed an
#   `i32` to a `ptr` param → `module verify failed: Call parameter type
#   does not match function signature`. Fix: prefix synthetic registers
#   `__t<N>` (kir_lower.kai), keeping them DISJOINT from source binders —
#   the reserved-synthetic convention the lowering already uses
#   (`__pcs_scr`, `_kai_acc_res`, `__self`). Native-path-only: the C
#   oracle lowers AST→C directly (never `lower_to_kir`), so selfhost
#   byte-id is unaffected. The whole `Call parameter type does not match`
#   family (5: map_basic/map_pipes/map_round_out/map_tree_basic + fx_basic)
#   closes; `jwt_encoder` (the 6th) now BUILDS + RUNS but DIVERGES on a
#   pre-existing decode bug the verify error masked (reclassified to
#   output-mismatch). The two Linux-only SIGSEGV gaps (`list_helpers`,
#   `list_zip3_scan`) PASS native on macOS post-fix but stay listed — the
#   ratchet is validated on Linux/CI and this lane cannot verify there;
#   if CI shows them passing consistently the ratchet prompts removal.
#   → 77 gaps remain — see docs/native-parity-gaps.md.
#
# Some gaps are FLAKY under native (intermittent SIGSEGV / pointer-bearing
# output): e.g. examples/stdlib/hex_basic.kai diverges ~50% of runs. Those
# are still gaps — intermittent parity is not parity — and are listed here.
# The ratchet re-verifies any "new gap" up to 3 times and only flags a
# regression when it diverges on EVERY attempt, so a sub-100% flaky fixture
# never fails CI at random.
#
# Do NOT add a fixture here to silence a regression — fix the backend.
# This list only records the gaps that predate the ratchet.
