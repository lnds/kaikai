# BinSerialize round-trip budget (issue #489).
#
# Wall-time ceiling for `tools/bench-binserialize-roundtrip.kai`,
# which runs 100 decodes over a 500-node / ~40 KB payload. The
# gate sits in Tier 2 (`make daily` → `test-binserialize-budget`)
# because perf telemetry belongs at the daily cadence, not at the
# per-PR Tier 1 gate.
#
# Format: a single integer = ceiling in milliseconds for the 100-
# decode total wall-time. Per-decode cost = ceiling / 100.
#
# PR #487 reported single-digit milliseconds per decode (~7 ms) on
# the post-Array[Byte] substrate. #485 reported ~19 s (O(N²)) on
# the pre-refactor cons-list. The ceiling below is 60 ms per decode
# (6000 ms total) — ~8× the reported baseline, leaving headroom
# for CI noise, ASAN runs, and incidental future GC / RC work.
#
# Ratchet down as future lanes improve, ratchet up only with a
# documented justification (commit message naming the lane that
# regressed and why the regression is acceptable).
6000
