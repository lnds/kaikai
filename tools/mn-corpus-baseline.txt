# mn-corpus-baseline.txt — the allowed-finding ratchet for
# tools/run-mn-corpus-determinism.sh.
#
#   <fixture-path> <backend> <issue> <one-line reason>
#
# Each line is a (fixture, backend) pair that does NOT reproduce its
# KAI_THREADS=1 behaviour above one thread, and that was ALREADY broken when
# this gate first walked the corpus. The contract:
#
#   - a pair failing now that is NOT listed here => FAIL (a regression).
#   - a listed pair that now reproduces N=1     => OK, with a prompt to
#                                                  remove the line.
#   - failing set == baseline                   => OK at baseline.
#
# So the queue can only shrink. The gate exists precisely because nothing
# was measuring this corpus above one thread; the queue below is what that
# first measurement found, not an allowance for new breakage. Every entry
# names an OPEN issue — the baseline is the bookmark, the issue is the work.
#
# The BUCKET is deliberately not part of the key: a baselined pair whose
# crash becomes a hang is the same known-broken pair, and flapping a
# required check on whichever bucket a race landed in would cost more than
# it buys. The issue carries that detail.
#
# The bar for adding a line is an OPEN ISSUE WITH A REPRO, never "it was red
# and I needed green". A fixture whose output LINE ORDER is genuinely
# unspecified does not belong here at all — declare it in
# tools/mn-corpus-order-dependent.txt, where the line multiset stays
# compared.
#
# These defects are TIMING- AND PLATFORM-DEPENDENT: a pair reproduces on one
# runner and not on another. This file is therefore the UNION over the
# platforms measured, and a run that does not reproduce an entry is not
# evidence the entry is fixed. The ratchet only ever fails on a pair that is
# absent, so platform variance costs detection, never a false red.
#
# First measured 2026-07-21 against main @ 1316a2ff. macOS/arm64 at N=4
# found 13 pairs over 8 fixtures; ubuntu-latest found a partly-overlapping
# set plus examples/effects/m8x_10_nursery_cancel_on_fail.kai, which macOS
# never reproduced. 1299 of 1312 fixture-backend pairs reproduced N=1
# exactly on the first sweep.
