# m8_4_cancel_caught: RESOLVED (parity lane, refs #622). The LLVM
# `handle` lowering omitted the m7a #6e setjmp/longjmp landing pad, so
# a clause that discards `resume` (Cancel.raise() forces it) could not
# unwind out of the body — the tail ran to completion (prints "never
# reached", returns 99) instead of yielding the clause value 42. The
# fix emits the setjmp pad in `llvm_emit_handle`, installs via
# `kaix_evidence_push_with_jmp`, and longjmps from the op site on a
# discard (`llvm_emit_op_dispatch`), mirroring emit_c.kai. The discard
# + body-result slots are `volatile` (C99 §7.13.2.1 mirror). Both
# backends now print 42, no "never reached". Coverage: this fixture +
# cancel_discard_minimal (single-frame) + cancel_discard_nested
# (three-frame unwind, ASAN guard). Removed from the skip list.

# m8_fiber_stack_overflow: RESOLVED (#718). The LLVM backend now emits
# scalar (unboxed) function signatures for UFns — `define i64
# @kai_deep(i64 %kair_n)`, mirroring C's `int64_t kai_deep(int64_t)` —
# instead of boxed `%KaiValue* @kai_deep(%KaiValue*)` with a `kaix_int`
# heap box per op. The body lowers fully raw (the EIf/EBlock-tail raw
# arms in `llvm_emit_kind_raw` were the missing piece), so each frame
# is now as small as C's and 50k frames hold in the 64 KiB private
# fiber stack on BOTH backends. Sibling of #706 (LLVM-TCO): both were
# the LLVM emitter dropping a C-path codegen optimization. emit_llvm
# consumes `UFnSig` from fnreg (no analysis change); selfhost stays
# byte-identical (kaic2b.c == kaic2c.c) and selfhost-llvm deterministic
# (s1.ll == s2.ll). Coverage: this fixture (unskipped) +
# examples/perceus/scalar_fn_sig_deep_recursion.kai. This was the LAST
# line in the #622 parity-bugs section — it is now empty, so #622 can
# close alongside #718 for Orongo (1.0).

# #618 — RESOLVED 2026-05-26 (llvm-parity-A)
# ============================================================
# The LLVM emitter dropped `\xNN` hex escapes in string literals:
# `json_byte_table()` in stdlib/encoding/json.kai is a `"\x01..\xff"`
# literal that `json_utf8_encode` slices into, so every JSON-decoded
# codepoint was corrupted under LLVM. Fixed in emit_llvm.kai
# `llvm_encode_body_loop` by decoding `\xNN` (advance cursor by 4).
# Removed: json_surrogate_decode, json_surrogate_encode (#618) and
# json_basic (#622) — all now byte-identical on C and LLVM.
