# native-selfhost-baseline.txt — the expected-fail ratchet for the
# native self-host gate.
#
# The single non-comment, non-blank line below is an integer: the count of
# `unbound register` subset-gap aborts the in-process libLLVM native backend
# (`kaic2 --emit=native`) emits when it compiles the COMPILER ITSELF
# (`stage2/main.kai`) — the one program no test exercises, so its KIR-node
# subset gap is otherwise silent. tools/test-native-selfhost-gate.sh gates:
#
#   - count  > baseline  => FAIL (regression — native lost ground: a change
#       made it reject a node it used to lower, or grew the compiler's use
#       of an already-unlowered node).
#   - count == baseline  => PASS (the documented expected-fail: the known
#       gap held steady; the self-compile produces no binary yet but got no
#       worse).
#   - count  < baseline  => PASS, with a prompt to lower this file to the
#       new count (ratchet tightens — mirror of tools/coverage-baseline.txt
#       / native-parity-baseline.txt).
#   - count == 0         => PASS, "native self-host ACHIEVED" — the gap is
#       closed and the gate should be upgraded to require the native object
#       to LINK + RUN.
#
# This gate measures only; closing the gap (driving the count to 0) lives in
# stage2/compiler/emit_native_fn.kai + kir_lower_*.kai.
#
# Breakdown of the current count by register name (context only; the gate
# counts the total):
#   n      x8   (let/var-bound value reads in arm bodies)
#   nm     x7
#   tyname x6
#   name   x2
#   eff    x2

25
