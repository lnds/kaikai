# mn-corpus-shards.txt — how tools/run-mn-corpus-determinism.sh is split
# across CI runners. One shard per line:
#
#   <shard-name> <dir> [<dir> ...]
#
# tools/mn-corpus-plan.sh turns this into the workflow's job matrix and, in
# the same pass, asserts the shards PARTITION the corpus: their union must
# equal KAI_CORPUS_DEFAULT_DIRS in tools/lib/corpus.sh and no directory may
# appear twice. A shard list that quietly stopped naming a directory would
# leave the gate green over a smaller corpus — the one failure mode a
# coverage gate must not have — so the split lives here, in one place, and
# is checked before a single fixture runs.
#
# The split balances fixture counts, not directories: examples/effects alone
# is a third of the corpus.

effects examples/effects
rc-and-concurrency examples/perceus examples/refinements examples/actors examples/llvm examples/spawn
stdlib-and-demos examples/packages examples/minimal examples/quickstart examples/stdlib examples/attributes examples/unstable demos
