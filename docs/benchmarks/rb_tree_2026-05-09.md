# RB-tree benchmark — 2026-05-09 (issue #403)

Re-measurement of the canonical 1M-insert red-black tree benchmark
(`examples/perceus/rb_tree_bench.{kai,sh}`) after the v0.45.0 / v0.46.0
nocturnal batch. Pins the post-fix wall, identifies the root-cause
commit of the wall regression flagged in #403, and records the
trade-off vs the pre-batch numbers.

## Root cause — PR #368 (commit `e093f1c`, closes #350)

Bisection across the 78 commits between `v0.44.1` and `v0.46.0`
narrows the regression to a single commit:

```
e093f1c fix(perceus): emit decref for matched scrutinee in tail-rebuild rotation arms (closes #350)
```

The commit adds `pcs_arm_drop_pass`, which inserts
`__perceus_drop(name)` between the body's statements and tail of
every match arm whose pattern introduces ≥ 2-use bindings. Pre-fix,
the `kai_incref` at every arm-binding extract site was unbalanced
— the read in source got a `__perceus_dup`, the consumer decref'd,
but the original extract ref was never dropped. Every fire of a
multi-use arm leaked the underlying child cell.

On the RB-tree workload the four rotation arms in `balance` extract
4–8 multi-use bindings each. Pre-fix the run leaked **21.96 M**
variant cells (`free_total = 6.67 M` against `alloc_total = 28.63 M`);
post-fix it frees **18.4 M more cells** (`free_total = 25.04 M`),
which is exactly the work the wall regression is paying for.

| Metric | Pre-#350 | Post-#350 | Δ |
| --- | ---: | ---: | ---: |
| `free_total` | 6 673 397 | 25 043 808 | +18 370 411 |
| `live_peak` | 21 956 449 | 4 354 112 | −80.2 % |
| RSS | 1 929 MB | 325 MB | −83.1 % |
| Wall | 2.27 s | 4.00 s | +1.76× |

The regression is **the structural cost of the leak fix**, not an
incidental codegen mistake. Every one of the +18.4 M frees represents
a cell that was leaking memory pre-fix.

## Mitigation in this lane (#403)

The `SExprStmt` lowering for `__perceus_drop(name)` shapes goes
through

```c
{ KaiValue *_ = kai_internal_drop(kai_x); kai_decref(_); }
```

`kai_internal_drop` already decrements `kai_x` and returns the
unit singleton (`rc == INT32_MAX`); the trailing `kai_decref(_)`
short-circuits on the singleton in the runtime, but the call +
branch still cost on every invocation. The issue-#350 pass emits
~18 M of these on this workload, so even a ~10 ns per-drop saving
adds up.

The fix specialises the `SExprStmt` emit path: when the wrapped
call is `__perceus_drop(_)`, swap the trailing `kai_decref(_)` for
`(void) _;`. Mirrors the existing tcrec rule-3 emission
(`{ KaiValue *_ = kai_internal_drop(kai_p); (void) _; }`).

The recovery is partial — wall drops from 4 197 ms to 4 032 ms
(−3.9 %), which is the entire avoidable share at this scope. The
bulk of the regression (~1 150 ms) is the cost of the +18.4 M
decrefs themselves and is structural.

## Why the lane stops here

Per the lane brief's failure-mode rule:

> If the regression is the cost of a correctness fix and cannot be
> reverted: STOP, document, close not-planned with explanation.

The remaining wall delta is structural cost of refcount work that
was being skipped illegally pre-#350. Reverting #350 reopens a
22 M-cell leak that drives RSS to 1.9 GB, which is a worse failure
mode than +1.1 s on a benchmark that is already 17× off its
DoD #4 target.

Closing the structural gap requires deeper Perceus work — typically
**reference-counting elision**: detect arm bindings that are read
exactly once via `__perceus_dup` and rewrite the dup-into-arg
pattern as a move (caller transfers its ref, no balanced incref/
decref pair generated). Out of scope for this lane; tracked
separately.

## Acceptance against #403

The lane brief sets a target of `≤ 2 800 ms` (5 % over the
pre-batch baseline). **Not met** — best post-fix wall is 4 032 ms.
The post-fix number is recorded as the new operational baseline;
the pre-#350 number was inflated by the 22 M-cell leak it was
shipping with and is not a meaningful target.

## Follow-ups

- **#403 closed not-planned** with this doc as the rationale.
- **New issue** for the structural fix: refcount-elision pass that
  collapses dup/drop pairs around single-use arm bindings.
- The benchmark fixture stays in the tree as the canonical Perceus
  measurement; future Perceus lanes should re-measure here.
