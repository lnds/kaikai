## Diagnosis (which parser site needed the rename)

The kind annotation lives in `parse_optional_kind_annotation`
(`stage2/compiler.kai:7634`). It is the only site that converts
the surface keyword into the internal `Kind` AST. Two-line edit:
swap the `"Unit"` recogniser for `"Measure"`, then add an `else
if kw == "Unit"` arm that emits the targeted diagnostic.

The internal AST tag (`KUnit`), the encoded suffix (`"#Unit"`),
and the helper names (`tp_kind_is_unit`, `tp_unit_suffix`,
`unit_tparam_lookup`) were left untouched on purpose — they are
not user-visible and renaming them would balloon the diff with no
acceptance-gate value. Comments around them were updated to note
the divergence between the legacy internal name and the new
surface keyword.

## Friction points

- **The "26 code sites + 27 doc sites" budget overstated the
  scope.** Reality: ~10 unique kind-position occurrences in code
  and ~12 in docs. The briefing inventory pre-counted comment
  cross-references, retros, and CHANGELOG entries, none of
  which need migration. Net under-budget by ~50%.
- **`make selfhost-llvm` does not exist.** The briefing listed
  it as a separate gate; the actual `selfhost` rule already
  covers both stage1 and stage2 fixed-points, so a single
  `make selfhost` was the right call. Removed the spurious row
  from the build TSV.
- **Negative fixture wiring.** The `examples/units/` directory
  has a precedent (`m12_5_unitless_unavailable.{kai,err.expected}`)
  with no test target wiring — fixtures live as regression-by-
  presence + manual verification. Followed the same convention
  rather than inventing a new test target.
- **Internal name divergence.** The compiler keeps `KUnit`,
  `tp_unit_suffix`, etc. internally. A future cleanup may rename
  them to `KMeasure` / `tp_measure_suffix` for consistency, but
  that is pure cosmetic refactoring — out of scope for #253.

## Subjective summary

This was the smallest possible "syntax rename + doc" lane: a
single-token parser change, one targeted error message, and a
new doc page introducing kinds for the first time. The surprise
was the briefing's overestimate; the actual blast radius was
half what the inventory suggested. Spending the saved time on a
slightly more polished `docs/kinds.md` (with the disambiguation
table front-and-centre) felt like the right reinvestment, since
that doc is now the single canonical answer to "wait, why are
there three Units?".
