## Scope as planned vs shipped

**Planned (owner's corrected scope):** the whole `docs/shape-kind-design.md`
design â€” kind `Shape`, the `Shape-App` formation rule, `[s: Shape]` tparams,

**Shipped and green (4 commits):**

1. **Kind `Shape` core.** `kind Shape : Structural` in the catalog (no `with`
   introducer â€” habitants derived from every arity-1 `type`). New `Ty` node
   `TyShapeApp(shape_var_id, arg)`; a `[s: Shape]` tparam applied as `s[A]`
   unifies first-order against a concrete `List[Int]` (`s := List, A := Int`)
   via a nullary constructor witness, and `apply_ty`/`subst_ty` reconstruct the
   concrete `TyListT`/`TyCon` so a resolved shape-app never reaches mono/codegen.
   `#Shape` tparam encoding + `KShape` kind discriminant. Formation validator
   rejects arityâ‰ 1 and composition `s[t[..]]` with clear diagnostics.

2. **Generalised fusion â€” the issue's actual goal.** A user container
   implementing a `Sequence[s: Shape]` protocol fuses its `map`-into-`foldl`
   pipeline: `xs | f |> foldl(i, c)` rewrites to a single `foldl(xs, i, (acc,x)
   => c(acc, f(x)))` â€” the model's own traversal runs once, the stage folds into
   the combiner, and the mapped structure is never built. Verified: the emitted
   C calls the impl's `foldl` and **never** its `map`; alloc count matches the
   source-build cost only (`test-shape-fusion-1200`). `list`/`Vec` keep their
   hand-written `map_foldl` fast-path, byte-identical (perceus-1134 goldens
   unchanged). Works on both the C and native backends.

3. **`: Functorial` / `axiom` surface.** `protocol P[s: Shape] : Functorial { }`
   (header, reusing the `kind K : Theory` spelling) and `impl P for T { } axiom
   Functorial` parse and validate against the closed theory catalog (`theory
   Functorial = { identity, fusion }` shipped). Stored as transparent
   `DProtoLaws`/`DImplAxiom` wrappers stripped to a `LawReg` before any Decl
   walker runs (the `lower_axioms` discipline).

4. **Autogenerated law-checks that run.** `law_checks.kai` synthesises, for each
   lawful non-`axiom` impl, `check` blocks over `T[Int]` witnesses enumerated
   from the type's own constructors (a nullary ctor seeds depth 0; a
   `Ctor(elem, recursive)` ctor extends it). Each law is verified by reducing
   both sides to a scalar via the impl's own `foldl`, so no container `Eq` is
   needed. The blocks are generated as source, re-parsed, and injected into the
   decl stream under `kai check` / `kai test`. Verified: a lawful `Chain` impl
   passes 6/6 (identity + fusion Ă— 3 witnesses); a tail-dropping `map` is caught
   by a counterexample on the depth-2 witness; the same impl marked
   `axiom Functorial` generates zero checks. Both backends. Gate:
   `test-shape-laws-1200`.

## Design decisions and alternatives

- **`TyShapeApp` node (asu consult).** Rejected head-variable-in-`TyCon`
  (touches every `name` comparison) and sentinel-name-in-`TyCon` (string-typing,
  contaminates the hot `TyCon~TyCon` unify path). The dedicated node isolates the
  concept: only the ~10 exhaustive `Ty` walkers get an arm; the `TyCon~TyCon`
  arm and the `name` field are untouched. Invariant: the `List` witness always
  reconstructs to `TyListT`, never `TyCon(_,"List",[a])`.

- **Shape-carrier dispatch.** A protocol quantified over a `Shape` is
  single-dispatch on the container; its ops take ordinary function arguments
  (`map(xs, f)`) whose types have no dispatch head. The dispatcher reads the head
  from the container arg and ignores function args â€” otherwise `args_head_names`'
  all-or-nothing rule vetoes the lookup and recursive impl-body op calls fall to
  a runtime dispatcher that panics.

- **Fusion by combiner composition, not KIR collapse (asu consult).** The
  architect's key finding: List fusion is not the KIR inliner collapsing
  `foldl(map(...))` â€” the typer rewrites directly to the hand-written
  `list.map_foldl`, a single TRMC traversal. The generic path reproduces that
  shape *through the model's own `foldl`* with a composed combiner, needing no
  new KIR machinery and no per-model hand-written combinator. Rejected: blind
  `foldl(map(...))` synthesis (leaves the intermediate) and KIR loop-collapse
  (doesn't exist for recursive containers).

- **Law-checks: enumerated witnesses + scalar reduction, not random
  property-based.** The random path needs a structural value-generator for user
  sum types plus a kaikai-callable `arbitrary_int` primitive (neither exists);
  enumerated witnesses from the type's constructors run deterministically with
  no new runtime. Comparison by `foldl`-to-scalar sidesteps deriving `Eq` for a
  parametric type (which the existing `derive_eq_impl` cannot do â€” it demands
  `Eq` on the tparam). Checks are generated as source and re-parsed rather than
  hand-built as AST: `check` bodies dispatch `map`/`foldl` to the impl by
  ordinary Shape-carrier dispatch, so a source block is both shorter and
  immune to slot-order drift in the AST types.

## Structural surprises

- **Selfhost byte-id caught two keyword-as-identifier bugs** (`impl` used as a
  `let`/param name) that kaic1 tolerated but the new kaic2 rejected â€” the gate
  earning its keep. Also caught the whole `Ty`/`Decl` exhaustiveness sweep: the
  new compiler enforces total matches, so `TyShapeApp` and the two law wrappers
  each needed arms across ~30 (`Ty`) and ~90 (`Decl`) sites.

- **Single-line `match` with a list-rest arm mis-parses** (`match xs { [] -> 0
  [h, ...t] -> h }` reads `0 [h,...]` as an index); arms must be on separate
  lines. Bit the fixtures twice.

- **The law-check blocker was a bootstrap-compiler name shadow, not a pipeline
  bug.** The first attempt read `TBSum` variant payloads as empty (`Link/0` for
  `Link(a, Chain[a])`). The cause was not a decl-stream transform (the args are
  present throughout â€” proved by calling the working `dump_variant` from the
  injection point) but a **kaic1 miscompilation**: a local binding named `args`
  shadowed a runtime symbol and read empty. Renaming it (`args` â†’ `vargs`) fixed
  the read. Two further kaic1 traps in the same code: a ctor pattern nested in a
  cons-pattern (`[CS(...), ...rest]`) panics "non-exhaustive match" even for a
  single-constructor type, and passing a variant's `[TypeExpr]` payload
  positionally into a matching helper corrupts it. The reliable idiom is
  `map`/`each` over the payload inside the same function that destructured the
  variant â€” exactly what `dump_variant` does.

## The native reg-slot desync (issue #1229) and its workaround

The most expensive surprise: the native-built self-hosted compiler SIGSEGV'd on
the CI runner while compiling *any* program, in `canon_tvars_ty`, decref'ing a
raw i64 (read from a variant slot) as a boxed pointer. It reproduced ONLY on the
GitHub runner â€” a local ubuntu:24.04 + LLVM 18 container self-compiled 30/30
clean, even with `MALLOC_CHECK_=3` / `MALLOC_PERTURB_` and under ASAN (the RC
allocator uses pools, not raw malloc, so ASAN's redzones don't cover the
mis-dropped cells). The C backend stayed byte-identical, so every C-only gate was
green â€” the crash was invisible to them.

Getting the stack required a temporary `gdb` diagnostic wired into the CI gate
(`gdb -batch -ex run -ex bt` around the crashing `--emit=c` call, `gdb` added to
the `build-native` apt install), read off the CI log, then reverted. Root cause
(asu): the #1110/#1201 class â€” adding the `TyShapeApp(Int, Ty)` arm to a hot
`match ty {}` walker introduces a kind-1 Int match-arm binder and shifts the
function's RSpec/alloca order, exposing a pre-existing reg-slot desync where a
binder's slot classification and perceus's RC plan diverge. NOT tag renumbering:
moving `TyShapeApp` to the end of the `Ty` type did not change the crash.

Workaround (this lane): route `canon_tvars_ty`'s `TyShapeApp` arm through a
helper taking `sv` as a plain param, so the kind-1 Int is not a match-arm
sub-binder the native classifier inspects. The underlying hole is #1229.

Second regression fixed here: the terminal-fold fusion probe
(`fuse_source_is_protocol_seq`) peeked the source head via a synth that binds in
the shared mutable substitution, but returned a bare Bool so the caller
continued from the stale fresh-tvar counter â€” the rewrite re-minted the peek's
ids and a stage lambda read its param at the peek's binding (`#1186`
"annotation needed"). Fixed by returning the post-peek state.

## Coverage gaps / follow-ups (for the owner to schedule)

- The native reg-slot desync #1229 â€” the lane workaround unblocks Shape but the
  lowering hole survives; any future arm with a kind-1 Int match-arm binder in a
  hot walker can re-trigger it.
- Witness enumeration covers the nullary + cons ctor shape (the fixture-gate
  target). Containers whose constructors do not fit `nullary` + `Ctor(elem,
  recursive)` yield no witness and get a build note rather than a silent pass;
  broadening the enumerator to arbitrary sum shapes is future work.
- `Vec` fusion via the generic path currently still uses the `map_foldl`
  fast-path (correct); the generic composed-combiner path fires only for
  non-`List`/`Vec` heads with a protocol `foldl` impl.
