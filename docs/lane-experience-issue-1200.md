# Lane experience — issue #1200: kind `Shape`

## Scope as planned vs shipped

**Planned (owner's corrected scope):** the whole `docs/shape-kind-design.md`
design — kind `Shape`, the `Shape-App` formation rule, `[s: Shape]` tparams,
first-order applicative unification, static `O(1)` dispatch, `laws Functorial` /
`axiom Functorial` surface, autogenerated per-impl law-checks, and generalised
pipe fusion (a user arity-1 container fusing its idiomatic `map`-into-`foldl`
pipeline) — as one continuous arc, no phasing stalls.

**Shipped and green (3 commits):**

1. **Kind `Shape` core.** `kind Shape : Structural` in the catalog (no `with`
   introducer — habitants derived from every arity-1 `type`). New `Ty` node
   `TyShapeApp(shape_var_id, arg)`; a `[s: Shape]` tparam applied as `s[A]`
   unifies first-order against a concrete `List[Int]` (`s := List, A := Int`)
   via a nullary constructor witness, and `apply_ty`/`subst_ty` reconstruct the
   concrete `TyListT`/`TyCon` so a resolved shape-app never reaches mono/codegen.
   `#Shape` tparam encoding + `KShape` kind discriminant. Formation validator
   rejects arity≠1 and composition `s[t[..]]` with clear diagnostics.

2. **Generalised fusion — the issue's actual goal.** A user container
   implementing a `Sequence[s: Shape]` protocol fuses its `map`-into-`foldl`
   pipeline: `xs | f |> foldl(i, c)` rewrites to a single `foldl(xs, i, (acc,x)
   => c(acc, f(x)))` — the model's own traversal runs once, the stage folds into
   the combiner, and the mapped structure is never built. Verified: the emitted
   C calls the impl's `foldl` and **never** its `map`; alloc count matches the
   source-build cost only (`test-shape-fusion-1200`). `list`/`Vec` keep their
   hand-written `map_foldl` fast-path, byte-identical (perceus-1134 goldens
   unchanged). Works on both the C and native backends.

3. **`laws` / `axiom` surface.** `protocol P[s: Shape] { } laws Functorial` and
   `impl P for T { } axiom Functorial` parse and validate against the closed
   theory catalog (`theory Functorial = { identity, fusion }` shipped). Stored
   as transparent `DProtoLaws`/`DImplAxiom` wrappers stripped to a `LawReg`
   before any Decl walker runs (the `lower_axioms` discipline).

**Not landed (a real blocker, surfaced for the owner — not a silent cut):**

- **Autogenerated law-checks that run.** The generation was implemented
  (`law_checks.kai`: enumerated `T[Int]` witnesses built from the type's
  constructors, laws compared by scalar reduction via the impl's own `foldl` so
  no container `Eq` is needed, injected as synthetic `DCheck`s that dispatch
  `map`/`foldl` to the impl). It reached the point of running under `kai check`
  but generated **0 checks**: at the injection point the container's `TBSum`
  variants read with **empty argument lists** (`Link/0` instead of `Link/2`),
  so no witness could be constructed — even though `--ast` on the same file
  shows the payloads intact and no pass emits `Var(_, [])`. The discrepancy
  between the raw parse (args present) and the read at the injection point
  (args empty) was not root-caused within the lane. Rather than ship a
  check-gen that silently emits zero checks (a half-feature), the code was
  removed and the blocker reported.

## Design decisions and alternatives

- **`TyShapeApp` node (asu consult).** Rejected head-variable-in-`TyCon`
  (touches every `name` comparison) and sentinel-name-in-`TyCon` (string-typing,
  contaminates the hot `TyCon~TyCon` unify path). The dedicated node isolates the
  concept: only the ~10 exhaustive `Ty` walkers get an arm; the `TyCon~TyCon`
  arm and the `name` field are untouched. Invariant: the `List` witness always
  reconstructs to `TyListT`, never `TyCon(_,"List",[a])`.

- **Shape-carrier dispatch.** A protocol quantified over a `Shape` is
  single-dispatch on the container; its ops take ordinary function arguments
  (`map(xs, f)`) whose types have no dispatch head. The dispatcher reads the head
  from the container arg and ignores function args — otherwise `args_head_names`'
  all-or-nothing rule vetoes the lookup and recursive impl-body op calls fall to
  a runtime dispatcher that panics.

- **Fusion by combiner composition, not KIR collapse (asu consult).** The
  architect's key finding: List fusion is not the KIR inliner collapsing
  `foldl(map(...))` — the typer rewrites directly to the hand-written
  `list.map_foldl`, a single TRMC traversal. The generic path reproduces that
  shape *through the model's own `foldl`* with a composed combiner, needing no
  new KIR machinery and no per-model hand-written combinator. Rejected: blind
  `foldl(map(...))` synthesis (leaves the intermediate) and KIR loop-collapse
  (doesn't exist for recursive containers).

- **Law-checks: enumerated witnesses + scalar reduction, not random
  property-based.** The random path needs a structural value-generator for user
  sum types plus a kaikai-callable `arbitrary_int` primitive (neither exists);
  enumerated witnesses from the type's constructors run deterministically with
  no new runtime. Comparison by `foldl`-to-scalar sidesteps deriving `Eq` for a
  parametric type (which the existing `derive_eq_impl` cannot do — it demands
  `Eq` on the tparam). This is the path that hit the empty-variant-args blocker.

## Structural surprises

- **Selfhost byte-id caught two keyword-as-identifier bugs** (`impl` used as a
  `let`/param name) that kaic1 tolerated but the new kaic2 rejected — the gate
  earning its keep. Also caught the whole `Ty`/`Decl` exhaustiveness sweep: the
  new compiler enforces total matches, so `TyShapeApp` and the two law wrappers
  each needed arms across ~30 (`Ty`) and ~90 (`Decl`) sites.

- **Single-line `match` with a list-rest arm mis-parses** (`match xs { [] -> 0
  [h, ...t] -> h }` reads `0 [h,...]` as an index); arms must be on separate
  lines. Bit the fixtures twice.

## Fixtures added

- `examples/sugars/shape_kind_dispatch` — List/Vec/user-type static dispatch.
- `examples/sugars/shape_kind_fusion` — user container + list both fuse.
- `examples/sugars/shape_kind_{arity,composition,scalar}_err` — formation and
  unification rejections with `.err.expected` goldens.
- `examples/perceus/shape_fusion_1200` + `test-shape-fusion-1200` — the
  measurable gate: emitted C calls the impl `foldl`, never `map`; alloc ==
  source-build only.

## Coverage gaps / follow-ups (for the owner to schedule)

- The autogenerated law-checks (blocker above). Needs: root-cause the
  empty-variant-args read at the injection point, then the generation code is
  ready to reinstate.
- `Vec` fusion via the generic path currently still uses the `map_foldl`
  fast-path (correct); the generic composed-combiner path fires only for
  non-`List`/`Vec` heads with a protocol `foldl` impl.

## Cost vs estimate

Larger than a single-feature lane: the arc spanned the typer (unification,
resolution, dispatch), the fusion pipeline, the parser (two clauses), and a
cross-file exhaustiveness sweep. The fusion — the load-bearing deliverable — and
the kind core landed cleanly; the law-check verification layer is where the arc
hit a genuine wall.
